nchoosekFS

Creates all combinations of n elements taken k at time or computes the number of such combinations.

Syntax

[out] = nchoosekFS(n,k)
[out] = nchoosekFS(v,k)

out is the number of all combinations if the first argument is a scalar n or it is a matrix with all combinations in the rows if the first argument is a vector v .

Description

This function is similar to nchoosek from the Statistics Toolbox, but in generating all combinations it is much faster and makes a more efficient use of memory.

    Note   The considerable performance gain is determined by the use of combsFS.

Example

First, try profiling the generation of 2118760 combinations.

v = 1:50; k = 4;

tic
for i=1:10, nchoosekFS(v,k); end
t_nchoosekFS = toc

tic
for i=1:10, nchoosek(v,k); end
t_nchoosek = toc

gain = t_nchoosek/t_nchoosekFS

nchoosekFS should be about 75 times faster than nchoosek.

In addition, in 32 bit platforms with 3Gbytes of RAM, if k is raised to 5 nchoosek will start swapping into virtual memory and, in such a case, performance will deteriorate drastically. nchoosekFS is generally not affected by this problem, as by default uses the int8 data type to store the numbers 1:k.


Now, just to show consistency with nchoosek, generates some combinations.

v = [1 2 3 4 5];

nchoosekFS(v,3)

ans =

1 2 3
1 2 4
1 2 5
1 3 4
1 3 5
1 4 5
2 3 4
2 3 5
2 4 5
3 4 5
